# VYTYLS NORWAY PROJECT - NEUTRAL SYSTEM SCHEMA

_(Cybernetic Acoustic Research & Performance Framework)_

---

## 1. ENTITY TYPES (NODES)

### A. SITE

**Definition:** A physical location with distinct acoustic, architectural, or linguistic properties.

**Examples:**

- Fjord (natural resonance field)
    
- Harbor / maritime infrastructure
    
- Urban vertical environment
    
- Cemetery / grave site
    
- Farm / ancestral land
    
- Transit system (ferry, port machinery)
    

**Attributes:**

- Geographic coordinates
    
- Acoustic profile
    
- Historical context (optional)
    
- Lineage relevance (optional constraint)
    

---

### B. CAPTURE

**Definition:** A method of extracting environmental data from a Site.

**Types:**

- Field audio recording
    
- Impulse Response (IR) capture
    
- Linguistic / phoneme sampling
    
- Environmental noise profiling (rain, wind, machinery)
    

**Attributes:**

- Capture method
    
- Dynamic range
    
- Temporal density
    
- Source Site reference
    

---

### C. TRANSLATION MODULE

**Definition:** A system component that converts captured data into behavioral input.

**Types:**

- Harmonic retuning (shared tritonic field)
    
- Granular deformation engine
    
- Cellular automaton driver (e.g., Conway’s Game of Life)
    
- Spectral / rhythmic mapping engine
    
- Probabilistic trigger system
    

**Attributes:**

- Input type compatibility
    
- Mapping rules
    
- Constraint parameters
    
- Output behavior range
    

---

### D. SYSTEM STATE

**Definition:** A bounded, condition-defined configuration of the performance system.

**Examples:**

- Field-specific performance state
    
- Harmonic field configuration
    
- Entropy window (bounded randomness)
    
- Stabilized live-performance mode
    

**Attributes:**

- Active Translation Modules
    
- Constraint values
    
- Environmental dependencies
    
- Performer interaction profile
    

---

### E. HUMAN AGENT

**Definition:** A performer interacting with and modulating the system.

**Roles:**

- Generative structure control (stochastic / harmonic)
    
- Sample deformation & temporal memory control
    
- Predictive synchronization with other agents
    

**Attributes:**

- Interface domain (Reaktor, Max/MSP, hardware)
    
- Action-response latency
    
- System familiarity
    

---

### F. OUTPUT

**Definition:** A material or experiential result of system operation.

**Types:**

- Live performance
    
- Multichannel audio recording
    
- Public sound archive
    
- Technical / artistic documentation
    
- Workshop / artist talk
    

**Attributes:**

- Format
    
- Duration
    
- Audience context
    
- Archival status
    

---

## 2. OPERATION TYPES (EDGES)

### 1. CONSTRAINT

**Definition:** Limits or channels system behavior or site selection.

**Examples:**

- Lineage → Site selection
    
- Harmonic field → sample admissibility
    
- Residency duration → system complexity
    

**Directionality:**  
Constraint → Entity

---

### 2. TRANSFORMATION

**Definition:** Converts one entity type into another functional form.

**Examples:**

- Site → Capture
    
- Capture → Translation Module input
    
- Environmental sound → behavioral driver
    

**Directionality:**  
Source → Transformed Entity

---

### 3. COUPLING

**Definition:** Bidirectional interaction between agents or systems.

**Examples:**

- Human ↔ Machine
    
- Performer ↔ Performer
    
- Translation Module ↔ System State
    

**Directionality:**  
Bidirectional

---

### 4. FEEDBACK

**Definition:** Output or performance alters future system behavior.

**Examples:**

- Performance → system retuning
    
- Archive → future project design
    
- Site return → revised capture strategy
    

**Directionality:**  
Output → Prior Entity
